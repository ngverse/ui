NgVerse styles are self-encapsulated, meaning that styling is done within the component's SCSS files rather than in a global theming file. Since you have full access to the component source code, you don't need to inspect elements or search through a global file to update styles. You can simply edit the SCSS file within the component itself.

However, NgVerse also adds a ngverse.scss file to your project. Let's break down what this file does.

ngverse.scss

Explanation

1. Normalize SCSS Import

The first line imports the normalize.scss file.

  • normalize.scss resets inconsistent browser styles. It uses this file for standardization.
  • If you already have a custom reset file (e.g., reset.scss), you can remove this import.
    Note: Small UI issues may occur if you use a different reset file. You might need to adjust component styles accordingly.

2. Angular CDK Overlay Import

The second line imports Angular CDK's overlay styles.

  • This is essential for floating panels (e.g., popovers, dropdowns) to display correctly.
  • If you are already using Angular Material or have imported this style manually in styles.scss, you can remove this import.

3. CSS Variables

Starting from the third line, CSS variables are defined. These variables are used throughout NgVerse components.

  • The main purpose of these variables is to maintain a coherent, default appearance across components.
  • If you do not have a custom design, the components will still look polished and consistent. You can update the variables to customize styles globally.
  • Each color variable follows this structure:
    • --color-name: Background color
    • --color-name-foreground: Text color

Automatic Setup

When you run the ng add ngverse schematics, the ngverse.scss file is generated and imported into your styles.scss file automatically.
